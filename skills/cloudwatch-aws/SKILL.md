# Querying AWS CloudWatch

## Listing log groups (CloudWatch Logs)
To find/list CloudWatch **log groups**, use **mcp_gf_cloudwatch_log_groups**:
- `pattern` — substring/wildcard filter (e.g. "gateway", "alb"); omit for all
- `region` — default us-west-2
Returns matching log-group names. Use this for "list / find / search log groups".

CloudWatch is a connected Grafana datasource. Query it with **mcp_gf_cloudwatch_metric**
— never tell the user to open the AWS Console.

## Region
Metrics for this stack live in **us-west-2**. If the user omits a region, default to
us-west-2. eu-west-2 / us-east-1 generally return nothing here.

## ALB / ELB errors and traffic  (namespace: AWS/ApplicationELB)
- `HTTPCode_ELB_5XX_Count`     — 5xx generated by the load balancer itself
- `HTTPCode_Target_5XX_Count`  — 5xx returned by the backend targets
- `HTTPCode_ELB_4XX_Count`     — client errors
- `TargetResponseTime`         — latency (use statistic Average or Maximum)
- `RequestCount`               — traffic volume
- `TargetConnectionErrorCount`, `RejectedConnectionCount`
Omit `dimensions` to see EVERY load balancer; series names are like
`app/<alb-name>/<id>`.

## Kafka / MSK by topic  (namespace: AWS/Kafka)
- `MessagesInPerSec`  — messages by topic (enhanced monitoring is enabled)
- `BytesInPerSec`, `BytesOutPerSec` — throughput by topic
- `GlobalTopicCount`  — total topic count
Omit `dimensions` to get every topic/broker; series names are `<brokerId> <topic>`.
`__consumer_offsets` is an internal Kafka topic. To expand by topic explicitly,
pass `dimensions={"Topic":"*"}`. Rows are per-topic-per-broker (each topic appears
once per broker).

## Other services
- EC2 → `AWS/EC2` (CPUUtilization, NetworkIn/Out)
- RDS → `AWS/RDS` (CPUUtilization, DatabaseConnections, FreeableMemory)
- Lambda → `AWS/Lambda` (Errors, Throttles, Duration, Invocations)

## Statistic
`Sum` for counts (errors, requests, messages); `Average`/`Maximum` for gauges
(latency, CPU). Default Sum.

## Answering
Summarize the top series by total, call out spikes with the time window, and note
that latency/throughput numbers are CloudWatch aggregates. Render results as a
markdown table (Series | Total | Peak | Last).
